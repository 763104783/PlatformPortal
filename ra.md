# 角色
- 你是资深全栈工程师，使用 vue3.x + springboot2.7.x + java1.8 开发web应用。
- 职责：负责web应用的ui设计+前端开发+后端开发

# 任务
实现一个赋能平台门户界面，负责根据需求开发业务app的前端、后端，并编写注释和接口文档。

# XX平台概念
- 该平台是一个基于模型驱动架构（MDA）的标准赋能平台，能够通过元数据模型-业务逻辑编排生成业务APP，类似当前主流的企业级低代码/aPaaS 平台，例如宜搭（阿里钉钉）、炎黄盈动 AWS PaaS
- 目标用户：业务app使用人员（通过导航找到业务app入口）、模型算法工程师（通过平台上传、编辑算法模型、数据模型、业务节点）、业务app搭建人员（零代码搭建轻应用）

# 核心场景
- 核心场景1：业务app使用人员，通过产品展示界面找到业务app入口，点击跳转到对应业务app使用；
- 核心场景2：模型算法工程师，通过后台管理界面管理某个算法功能模型、数据实体模型，实现模型的新增、更新、删除、查看；
- 核心场景3：业务app搭建人员，通过操作工作台界面，通过 “配置元数据 + 可视化编排” 实现业务应用的快速搭建；

# XX平台技术分层
- 1：数据连接层，统一数据底座，屏蔽存储与系统差异
- 2：元数据模型层，平台业务骨架，所有要素的可视化定义
- 3：业务编排层，平台调度大脑，可视化串联所有能力
- 4：应用交付层，最终业务产出，面向终端用户

# XX平台的数据结构
- 平台信息数据结构：平台名称、简介、logo、制作公司名称、版本号；
- 产品信息数据结构：分为交付应用、业务流程、模型三种类型，具体如下：
- 产品信息数据结构1-应用类型数据结构：枚举量，应用类型名称、简介；
- 产品信息数据结构2-业务流程模板数据结构：枚举量，业务流程模板名称、简介；
- 产品信息数据结构3-数据实体模型数据结构：实体量，数据实体模型的名称、简介、制作人、制作时间、版本号；
- 产品信息数据结构4-算法能力模型数据结构：实体量，算法能力模型的名称、简介、制作人、制作时间、版本号；
- 产品信息数据结构5-业务流程数据结构：实体量，示例业务流程的名称、业务流程模板值（业务流程模板1、业务流程模板2、业务流程模板3）、简介、制作人、制作时间；
- 产品信息数据结构6-交付应用信息数据结构：实体量，业务应用app的名称、应用类型值（应用类型1、应用类型2...）、业务流程值（业务流程1、业务流程2、业务流程3）、网址链接、功能简介、封面图片、版本号、详细介绍等必要信息；

# XX平台界面设计
- 平台分为门户欢迎界面、公司产品业绩宣贯界面、产品展示界面、操作工作台界面、后台管理界面
- 第一层：采用门户欢迎界面：展示XX平台的名称、logo、公司名称、产品展示界面入口、公司产品业绩宣贯界面入口。采用左右布局，左边是平台名称、logo等，右边是入口，突出左边即突出平台的名称；
- 第二层1：公司产品宣贯界面：图表展示用户生态、产品体系、模型体系、数据体系、知识产权等方面成绩；
- 第二层2：产品展示界面：采用侧边栏设计。左边是导航侧边栏，右边是业务区，业务区上边是检索框、分类卡片等；
- 第二层2：产品展示界面的侧边栏采用树形结构分为：应用交付、业务编排、模型管理三个父节点。侧边栏数据结构如下：
（父节点1）应用交付-（子节点1、2、3）应用类型1、应用类型2、应用类型3
（父节点2）业务编排-（子节点1、2、3）业务类型1、业务类型2、业务类型3
（父节点1）模型管理-（子节点1、2）数据实体、算法能力
- 第二层2-1：产品展示-应用交付界面，采用卡片布局+列表布局（两种布局可切换）展示业务app入口，点击卡片可以跳转到对应app；
- 第二层2-1：产品展示-业务编排界面，采用卡片布局+列表布局（两种布局可切换）展示已有业务流程，点击卡片可以查看具体业务流程；
- 第二层2-3：产品展示-模型管理界面，采用列表布局展示已有模型，点击可以查看具体模型信息；
- 第二层3：操作工作台界面产品:采用侧边栏设计。左边是导航侧边栏，右边是业务区，业务区上边是检索框等；
- 第二层3：产品展示界面的侧边栏采用树形结构分为：平台信息、应用交付、业务编排、模型管理(数据实体模型、算法能力模型两个子节点)三个父节点。
- 第二层4：后台管理界面，采用侧边栏设计。左边是导航侧边栏，右边是业务区，业务区上边是检索框。
- 第二层4：后台管理界面的侧边栏采用树形结构分为：平台信息、应用交付、业务编排、模型管理(数据实体模型、算法能力模型两个子节点)三个节点。

# XX平台使用场景操作流程
- 使用场景1：业务app使用人员：进入门户欢迎界面-点击产品展示界面入口并进入-点击侧边栏的应用交付-应用类型1，在右边业务区点击某个应用卡片并跳转；
- 使用场景2：模型算法工程师：进入门户欢迎界面-点击后台管理入口并进入-点击侧边栏的模型管理-点击数据实体模型/算法能力模型，在右边业务区编辑某条数据；
- 使用场景3：平台管理人员：进入门户欢迎界面-点击后台管理入口并进入-点击侧边栏的各类节点，在右边业务区编辑某条数据；

# XX平台的界面功能需求
- 支持用户权限管理，管理员、普通用户两种角色，管理员比普通用户多 编辑数据、管理普通用户的权限；
- 支持管理员修改平台信息数据结构。
- 支持在管理员新增、编辑、删除（批量）、修改应用类型数据实例
- 支持在管理员新增、编辑、删除（批量）、修改业务流程模板数据实例
- 支持在管理员新增、编辑、删除（批量）、修改数据实体模型数据实例
- 支持在管理员新增、编辑、删除（批量）、修改算法能力模型数据实例
- 支持在管理员新增、编辑、删除（批量）、修改业务流程数据实例
- 支持在管理员新增、编辑、删除（批量）、修改交付应用信息数据实例
- 支持在业务app导航界面上点击链接跳转到各个业务app，使用打开新浏览器的方式跳转
- 支持在业务app导航界面上按业务app的信息筛选、查寻业务app链接，例如名称、类型、简介等
- 支持管理员新增、编辑、删除用户生态、产品体系、模型体系、数据体系、知识产权等数据
- 支持在公司产品宣贯界面上查看用户生态、产品体系、模型体系、数据体系、知识产权概览数据，点击后查看详细数据。
- 支持以统计已经收录的业务app数据、用户数据，并以图表形式展示，例如收录数量、用户数量、点击次数等
