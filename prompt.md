# 角色
你是资深全栈工程师，使用 vue3.x + springboot2.17.x + java1.8 开发web应用。

# 任务
实现一个名字叫“XX平台”的多web应用导航的web应用开发，职责：负责web应用的ui设计+前端开发+后端开发

# XX平台概念
- 该平台是一个我公司已开发的多个web应用的“导航界面”、“宣传手册”，能够通过该应用进入该各个web应用，同时能够展示我司开发的web应用的多领域、成果丰富、业绩好的特点；
- 目标用户：对我司感兴趣的潜在客户、我司领导
- 核心场景：销售向客户介绍我司产品，或者用户通过该应用快速找到业务应用app并进入使用

# XX平台界面设计
- 第一层：采用门户欢迎界面：展示XX平台的名称、logo、公司名称、web应用导航界面入口、公司产品宣贯界面入口；
- 第二层1：公司产品宣贯界面：图表展示用户生态、产品体系、模型体系、数据体系、知识产权等方面成绩；
- 第二层2：web应用导航界面：展示各种web应用入口（卡片布局+列表布局）

# XX平台界面风格
- 主题颜色为紫色，采用紫荆花的紫色
- 风格为科技+军事实战风格

# XX平台功能需求
- 支持用户权限管理，管理员、普通用户两种角色，管理员比普通用户多 编辑数据、管理普通用户的权限；
- 支持在管理员新增、编辑、删除web应用链接，web应用的展示信息包括web应用的名称、功能简介、类型、封面图片、版本号、详细介绍等必要信息
- 支持在web应用导航界面上点击链接跳转到各个web应用，使用打开新浏览器的方式跳转
- 支持在web应用导航界面上按web应用的信息筛选、查寻、排序web应用链接，例如名称、类型、简介等
- 支持管理员新增、编辑、删除用户生态、产品体系、模型体系、数据体系、知识产权等数据
- 支持在公司产品宣贯界面上查看用户生态、产品体系、模型体系、数据体系、知识产权概览数据，点击后查看详细数据。
- 支持以统计已经收录的web应用数据、用户数据，并以图表形式展示，例如收录数量、用户数量、点击次数等
- 支持管理员修改门户欢迎界面的平台名称、logo、公司名称；修改网页的底图。

# XX平台开发需求
- 数据存入sqlite轻量数据库
- 代码写好注释
- 给出接口说明，支持后续人工开发、调试

# 验收条件
- 实现平台功能需求
- 提供前端项目代码、后端项目代码
- 提供接口说明文档
