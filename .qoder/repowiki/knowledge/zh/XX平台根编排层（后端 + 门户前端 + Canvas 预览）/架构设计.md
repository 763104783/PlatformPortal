根目录不承载业务逻辑，仅作为跨子模块的编排与契约层：
- `API.md` 是 backend 与 frontend_portal 共同遵守的 REST 契约，定义 `/api` 前缀、统一 `{code,message,data}` 响应体、Token 认证头以及认证/用户/应用/分类/宣贯/配置/统计七大模块共 26 个端点；frontend_portal 通过 Vite dev proxy 将 `/api` 转发到 backend 8080，实现仅 HTTP 耦合的双进程架构。
- `Canvas.html` + `xx-platform-report.canvas.tsx` 构成 Canvas 预览入口，使用 importmap 把 `qoder/canvas`、`lingma/canvas`、`aicoding/canvas`、`cursor/canvas` 等品牌别名全部重定向到同一 `/canvas-sdk.js`，由 `mountCanvas` 加载模块并渲染完成度报告，使同一份 Canvas 代码在多个 IDE 品牌下复用。
- `prompt.md` 作为需求基线约束角色、界面分层、紫色科技风格、SQLite 存储与验收条件，backend 与 frontend_portal 均以此为源独立实现。
- 运行时启动遵循文档约定：先 `cd backend && mvn spring-boot:run`，再 `cd frontend_portal && npm run dev`，两者独立进程、仅通过 HTTP 耦合。
