## 1 引言

### 1.1 编写目的

本文档旨在明确XX平台的功能需求、非功能需求及约束条件，作为系统设计、开发、测试和验收的基准依据。

### 1.2 项目背景

XX平台是一个基于**模型驱动架构（MDA）**的标准赋能平台，能够通过"元数据模型 + 业务逻辑编排"生成业务APP，类似当前主流的企业级低代码/aPaaS平台（如阿里宜搭、炎黄盈动 AWS PaaS）。

### 1.3 术语定义

| 术语 | 说明 |
|------|------|
| MDA | Model-Driven Architecture，模型驱动架构 |
| 元数据模型 | 描述业务要素结构的数据定义 |
| 业务编排 | 通过可视化方式串联业务能力形成业务流程 |
| 交付应用 | 最终面向终端用户的业务APP |
| 数据实体模型 | 描述业务数据结构的模型定义 |
| 算法能力模型 | 描述算法功能的模型定义 |
| 业务流程模板 | 预定义的业务流程类型枚举 |
| 应用类型 | 预定义的应用分类枚举 |

### 1.4 参考资料

- 项目需求文档：`ra.md`
- 技术栈：Vue 3.x + SpringBoot 2.7.x + Java 1.8 + SQLite

## 2 系统概述

### 2.1 平台定位

XX平台是一个多角色、多场景的赋能平台门户系统，为业务应用的使用、管理和搭建提供统一入口。

### 2.2 目标用户

| 角色 | 描述 | 核心诉求 |
|------|------|----------|
| 业务APP使用人员 | 最终业务用户 | 通过导航快速找到并使用业务APP |
| 模型算法工程师 | 技术人员 | 上传、编辑、管理算法模型和数据模型 |
| 业务APP搭建人员 | 业务/技术人员 | 零代码搭建轻量业务应用 |
| 平台管理员 | 运维管理人员 | 管理平台信息、用户权限、所有数据 |

### 2.3 技术分层架构

```
┌─────────────────────────────────────────┐
│  第4层：应用交付层（面向终端用户的业务APP）  │
├─────────────────────────────────────────┤
│  第3层：业务编排层（可视化串联所有能力）      │
├─────────────────────────────────────────┤
│  第2层：元数据模型层（所有要素的可视化定义）  │
├─────────────────────────────────────────┤
│  第1层：数据连接层（统一数据底座）           │
└─────────────────────────────────────────┘
```

## 5 核心使用场景

### 场景1：业务APP使用人员

```
进入门户欢迎界面 → 点击"产品展示"入口 → 点击侧边栏"应用交付-应用类型X" 
→ 在业务区找到目标应用卡片 → 点击卡片 → 新窗口打开对应业务APP
```

### 场景2：模型算法工程师

```
进入门户欢迎界面 → 点击"后台管理"入口 → 点击侧边栏"模型管理-数据实体模型/算法能力模型" 
→ 在业务区新增/编辑/删除模型数据
```

### 场景3：平台管理人员

```
进入门户欢迎界面 → 点击"后台管理"入口 → 点击侧边栏各节点 
→ 在业务区管理平台信息、应用、流程、模型等数据
```

## 6 约束与假设

### 6.1 技术约束

- 前端：Vue 3.x + Vite + Element Plus + ECharts
- 后端：SpringBoot 2.7.x + Java 1.8 + MyBatis-Plus
- 数据库：SQLite（轻量嵌入式）
- 部署：前后端分别启动

### 6.2 假设

- 系统为内网部署，无外网访问需求
- 用户规模较小（<100人），无需分布式架构
- 文件上传大小限制为5MB
