## 1 引言

### 1.1 编写目的

本文档描述XX平台的整体架构设计、模块划分、技术选型及模块间交互关系，为详细设计和编码实现提供指导。

### 1.2 设计原则

- **前后端分离**：前端SPA + 后端REST API，通过JSON通信
- **分层架构**：Controller → Service → Mapper 三层结构
- **模块化设计**：按业务域划分独立模块，低耦合高内聚
- **统一规范**：统一响应格式、统一异常处理、统一认证机制

## 4 认证与权限设计

### 4.1 认证流程

```
用户登录 → 后端验证用户名/密码 → 生成Token(UUID) → 返回Token
     ↓
后续请求 → 请求头携带 Authorization: {token}
     ↓
后端拦截 → 验证Token有效性 → 获取用户信息 → 校验角色权限
```

### 4.2 权限矩阵

| 功能 | 管理员(ADMIN) | 普通用户(USER) |
|------|:---:|:---:|
| 浏览产品展示 | ✓ | ✓ |
| 浏览业绩宣贯 | ✓ | ✓ |
| 查看模型详情 | ✓ | ✓ |
| 跳转业务APP | ✓ | ✓ |
| 编辑平台信息 | ✓ | ✗ |
| 管理应用/流程/模型 | ✓ | ✗ |
| 管理宣贯数据 | ✓ | ✗ |
| 管理用户 | ✓ | ✗ |
| 访问后台管理 | ✓ | ✗ |
| 访问操作工作台 | ✓ | ✗ |

## 5 前后端交互设计

### 5.1 统一响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

| code | 含义 |
|------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未认证/Token过期 |
| 403 | 权限不足 |
| 500 | 服务器内部错误 |

### 5.2 分页请求/响应约定

**请求参数：**
- `page`：页码（从1开始）
- `size`：每页条数（默认10）

**响应格式：**
```json
{
  "code": 200,
  "data": {
    "records": [...],
    "total": 100,
    "page": 1,
    "size": 10
  }
}
```

### 5.3 文件上传

- 接口：`POST /api/config/upload`
- 格式：`multipart/form-data`
- 限制：图片文件，≤5MB
- 返回：文件访问路径

## 7 关键设计决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 数据库 | SQLite | 轻量免部署，适合小规模内网系统 |
| 认证方式 | 内存Token | 简单高效，无需Redis等中间件 |
| 前端分页 | 后端分页为主 | 数据量增长后性能可控 |
| 文件存储 | 本地磁盘 | 简单直接，无需对象存储 |
| 枚举管理 | 数据库表 | 支持动态增删，比代码枚举灵活 |
